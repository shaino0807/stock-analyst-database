## 踩坑與修復

- Windows PowerShell 5.1 讀寫中文 literal 容易出現 mojibake。資料庫顯示名稱改用 Unicode codepoint 組字，避免產生亂碼標題。
- `yt-dlp` 反覆警告沒有 supported JavaScript runtime。共用工具已自動尋找 Node.js 並補上 `--js-runtimes node:<path>`。
- YouTube 標題與字幕包含中文時，PowerShell 預設 cp950 可能讓 yt-dlp stdout 解碼失敗。已在呼叫 yt-dlp 時暫時設定 `PYTHONIOENCODING=utf-8` 與 UTF-8 console output。
- 容量估算一開始在 strict mode 下讀取不存在的 `requested_formats`、`filesize`、`filesize_approx` 會失敗。已改成透過 `.PSObject.Properties[...]` 檢查屬性存在後再讀。
- metadata 缺口不能只看檔名，因為輸出檔名含標題與日期。後續以 `.info.json` 內的 `id` 比對影片 URL，避免把已抓到的 metadata 誤判為缺漏。
- GitHub Pages 與 GitHub Actions 不能只看本地檔案完成，還需要確認 remote、Actions conclusion 與公開 URL 回應。
