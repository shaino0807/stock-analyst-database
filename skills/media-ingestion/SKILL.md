# Media Ingestion

## Overview

Use the root `scripts/` commands as deterministic entry points for media collection. Prefer subtitles and metadata before full video downloads when the task is research, summarization, finance-news monitoring, or knowledge-base ingestion.

## Safety Rules

- Use only URLs the user owns, is authorized to process, are public-domain/Creative Commons where allowed, or are being used under a defensible limited research workflow.
- Do not help re-upload or redistribute someone else's video or audio.
- For finance/news workflows, collect title, description, metadata, subtitles, and short cited references first; avoid downloading full videos unless the user has a clear legitimate reason.
- For AI animation, VFX, or music analysis, use media only for structural reference such as pacing, shots, captions, and audio sections. Do not copy a creator's protected expression, living-artist identity, or protected IP.
- Keep outputs in `outputs/media/`; do not commit downloaded media artifacts.

## Commands

Run from the project root.

```powershell
.\scripts\download-video.ps1 -Url "https://example.com/video"
.\scripts\extract-audio.ps1 -Url "https://example.com/video"
.\scripts\extract-subtitles.ps1 -Url "https://example.com/video" -IncludeAutoGenerated
.\scripts\metadata-only.ps1 -Url "https://example.com/video"
```

`download-video.ps1` defaults to `-MaxHeight 1080` to avoid accidental 4K downloads. Use `-MaxHeight 360` for lightweight tests or `-MaxHeight 2160` only when a high-resolution local reference is actually needed.

Subtitle extraction defaults to `zh-TW,zh-Hant,zh-Hans,en` to avoid requesting every auto-translated caption track from YouTube. Pass `-SubtitleLanguages "zh-TW,en"` or another explicit list when the workflow needs different languages.

Batch mode:

```powershell
.\scripts\metadata-only.ps1 -BatchFile .\batch-list.txt
.\scripts\extract-subtitles.ps1 -BatchFile .\batch-list.txt -IncludeAutoGenerated
```

Use `-DryRun` before large jobs to inspect the generated `yt-dlp` command. Use `-AllowPlaylist` only when intentionally processing a playlist; scripts default to `--no-playlist`.

## Tool Resolution

Scripts resolve tools in this order:

1. `YTDLP_PATH` / `FFMPEG_PATH` environment variables
2. `yt-dlp` / `ffmpeg` on `PATH`
3. The known local global install under `C:\Users\shaino\Documents\Codex\2026-06-25\yt-dlp\tools\media\`

## Output Contract

- Videos: `outputs/media/videos/`
- Audio: `outputs/media/audio/`
- Subtitles: `outputs/media/subtitles/`
- Metadata JSON: `outputs/media/metadata/`

Use metadata JSON and subtitle/audio outputs as the handoff to downstream agents for transcription, summaries, script analysis, clipping plans, tagging, and knowledge-base insertion.
